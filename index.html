Flujo de aprobación

  1. Solicitud del cliente ingresa (correo / WhatsApp / llamada).
  2. Andrea (Pricing) cotiza en este cotizador → obtiene margen y estado.
  3. Si "Autónomo": envía al cliente directamente (máx. 2 horas).
  4. Si "Requiere aprobación": envía a Aldo por WhatsApp con contexto (cliente, volumen, histórico).
  5. Aldo responde APROBADO / AJUSTAR en máx. 4 horas hábiles.
  6. Se guarda la cotización en el REGISTRO con su estado.
  7. Lunes se revisa DASHBOARD en reunión de pipeline.

PASO 3 — Después de crear, copia el ID de aplicación (client ID) y el ID de directorio (tenant ID) y pégalos en los campos de arriba.

PASO 4 — En la app registrada → Permisos de API → Agregar permiso → Microsoft Graph → Permisos delegados → agrega: Sites.ReadWrite.All → Conceder consentimiento de administrador

PASO 5 — En tu sitio SharePoint → Nueva → Lista → nombre exacto: CotizacionesPricing. Las columnas se crean automáticamente la primera vez que alguien guarda una cotización.

PASO 6 — Cada colaborador abre el HTML en SharePoint, va a la pestaña Registro, llena Client ID, Tenant ID y URL del sitio, y hace clic en "Conectar con Microsoft 365". Sus credenciales se guardan en su navegador.
